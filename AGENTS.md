## Deployment Workflow

CRITICAL: After completing any feature, fix, refactor, or update, you must immediately create a git commit and push it to the tracked remote branch.

Do not stop after only editing files. A task is not complete until changes are committed and pushed.

Required workflow for every completed code change:
1. Review the diff.
2. Create a clear commit.
3. Push to the current remote branch.

Every push triggers an automatic deployment on Vercel.
